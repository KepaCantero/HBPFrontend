Getting Started

Yo {{appTitle}}!

Your application has been generated. Now is the time to start the real work.

You will find below a list of steps to take in order to get started with the HBP/BBP modules and pointers to other commonly used module documentation.

BBP Config

angular-bbp-config module is required by most of the other HBP modules to retrieve specific environment variables which ranges from REST services endpoint URL to feature activations.

It is up to every application to retrieve this information before the angular application is bootstraped.

The current template asynchronously load ./config.json before starting the application. The code that does that is located at the end of app/scripts/app.js.

The content of config.json should not be deployed real server. Instead, you should rely and customize the following Puppet Hiera data file: infra/bbp-puppet-modules.git/hieradata/hostgroup/webapplication/platform.yaml

Authentication using OIDC Client

bbp-oidc-client module is used to authenticate against the HBP OIDC servers.

You can configure its behavior in app/scripts/app.js.

HBP Common

angular-hbp-common is a set of common resources to achieve an uniform look & feel amongs all HBP web applications.

Common CSS style are described in the styleguide.

Common AngularJS directive have to be documented. At this point, you can browse the sources in bower_components/angular-hbp-common/angular-hbp-common.js
