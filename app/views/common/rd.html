Robot Designer

The Robot Designer development will start soon.

The Robot Designer will allow a user to easily set up a virtual robot for a given simulated experiment. The word "easily" is here very important since the user may be a neuroscientist, and unfamiliar with robotics details. The robot shall be integrated into an environment later on.

In the first version of the Robot Designer, to be completed at the end of the ramp-up phase, the Designer will provide the user with pre-assembled robots from a "robot library", or from imported files for quick customisation.

The team is currently implementing two robots for the platform: The Clearpath Husky robot and a biologically inspired mouse model. The Husky is a realistic model of a mobile robot distributed by a company called Clearpath. It will therefore be a good demonstrator for the transition from simulated to physical robotics experiments, which are planned for the FPA phase of HBP. The mouse model caters to the needs of neuroscientists and the plans to run behavioural experiments in simulation.

Meanwhile...

The neurorobotics platform is built on top of Gazebo . Any robots created for Gazebo is therefore compatible with the platform. Gazebo uses the SDF format. If you want your robots to be available, you need to upload the corresponding SDF file(s) and assets through the portal document browser (). In order for it to be recognised by the platform, the main sdf file needs to be edited:

  • The content type should be changed to nrp/robot
  • A custom attribute nrpDescription with a string containing the description of the robot needs to be added.
  • A custom attribute nrpName with a string containing the name of the robot needs to be added,
  • An optional attribute thumbnail with the uuid to an image uploaded to the portal representing the robot can be added.
